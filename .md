# **1. Write in detail about Data Types in Java**



## **1. Introduction**

Java is a **strongly typed, object-oriented, and platform-independent programming language**.
To handle different kinds of data in an efficient and secure manner, Java provides a well-defined system of **data types**.
Data types play a **crucial role** in deciding **memory allocation**, **type safety**, **range of values**, and **operations** that can be performed on data.



## **2. Definition**

A **data type** in Java is a classification that specifies:

* The **type of data** a variable can store
* The **amount of memory** allocated
* The **range of values**
* The **operations** permitted on the data



## **3. Importance of Data Types**

Data types are important because they:

* Help in **efficient memory management**
* Prevent **type mismatch errors**
* Improve **program reliability**
* Ensure **platform independence**
* Enable **compile-time error checking**



## **4. Classification of Data Types in Java**

Java data types are classified into **two major categories**:

1. **Primitive Data Types**
2. **Non-Primitive (Reference) Data Types**



## **5. Primitive Data Types**

### **5.1 Definition**

Primitive data types are **basic data types** that store **simple values directly in memory**.
They occupy a **fixed amount of memory** and are **predefined by Java**.



### **5.2 List of Primitive Data Types**

| Data Type | Size    | Default Value | Range                |
|-----------|---------|---------------|----------------------|
| `byte`    | 1 byte  | 0             | –128 to 127          |
| `short`   | 2 bytes | 0             | –32,768 to 32,767    |
| `int`     | 4 bytes | 0             | –2³¹ to 2³¹–1        |
| `long`    | 8 bytes | 0L            | –2⁶³ to 2⁶³–1        |
| `float`   | 4 bytes | 0.0f          | 6–7 decimal digits   |
| `double`  | 8 bytes | 0.0           | 15–16 decimal digits |
| `char`    | 2 bytes | `\u0000`      | Unicode characters   |
| `boolean` | 1 bit   | false         | true / false         |



### **5.3 Integer Data Types**

Integer data types are used to store **whole numbers**.
Among them, `int` is the **default integer type**, and `long` is used for **very large values**.

#### **Program: Integer Data Types**

```java
class IntegerDataTypes {
    public static void main(String[] args) {
        int age = 18;
        long population = 1400000000L;

        System.out.println("Age: " + age);
        System.out.println("Population: " + population);
    }
}
```



### **5.4 Floating-Point Data Types**

Floating-point data types are used to store **decimal values**.
`double` is the **default floating-point type**, while `float` must end with `f`.

#### **Program: Floating-Point Data Types**

```java
class FloatingPointTypes {
    public static void main(String[] args) {
        float marks = 85.5f;
        double pi = 3.14159;

        System.out.println("Marks: " + marks);
        System.out.println("Value of Pi: " + pi);
    }
}
```



### **5.5 Character Data Type**

The `char` data type stores **a single Unicode character** and occupies **2 bytes of memory**.

#### **Program: Character Data Type**

```java
class CharacterType {
    public static void main(String[] args) {
        char grade = 'A';

        System.out.println("Grade: " + grade);
    }
}
```



### **5.6 Boolean Data Type**

The `boolean` data type stores **logical values** and is mainly used in **decision making**.

#### **Program: Boolean Data Type**

```java
class BooleanType {
    public static void main(String[] args) {
        boolean isPassed = true;

        System.out.println("Result: " + isPassed);
    }
}
```



### **5.7 Features of Primitive Data Types**

* Faster execution
* Fixed memory size
* Stores actual value
* Memory efficient
* Do not support object-oriented features



## **6. Non-Primitive (Reference) Data Types**

### **6.1 Definition**

Non-primitive data types store the **reference (address)** of objects instead of storing actual values.



### **6.2 Types of Non-Primitive Data Types**

* String
* Array
* Class
* Interface
* Object



### **6.3 Program: String Data Type**

```java
class StringType {
    public static void main(String[] args) {
        String name = "Krishna";

        System.out.println("Name: " + name);
    }
}
```



### **6.4 Program: Array Data Type**

```java
class ArrayType {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3, 4};

        for (int i = 0; i < numbers.length; i++) {
            System.out.println(numbers[i]);
        }
    }
}
```



### **6.5 Features of Non-Primitive Data Types**

* Dynamic memory allocation
* Can store multiple values
* Support object-oriented programming
* Memory stored in **heap**
* Can have methods



## **7. Difference Between Primitive and Non-Primitive Data Types**

| Primitive      | Non-Primitive    |
| -- | - |
| Stores value   | Stores reference |
| Fixed size     | Dynamic size     |
| Faster         | Slower           |
| Predefined     | User-defined     |
| Cannot be null | Can be null      |



## **8. Java Wrapper Classes**

Java provides **wrapper classes** to convert primitive data types into objects.

| Primitive | Wrapper Class |
|  | - |
| int       | Integer       |
| char      | Character     |
| boolean   | Boolean       |

**Uses of Wrapper Classes:**

* Used in **Collections**
* Enables **object representation** of primitives



## **9. Advantages of Java Data Types**

* Platform independent
* Secure and reliable
* Efficient memory utilization
* Reduces runtime errors
* Improves program clarity



## **10. Conclusion**

Data types form the **foundation of Java programming**.
Understanding both **primitive** and **non-primitive** data types helps programmers write **efficient, secure, and error-free Java programs**.




# **Write in detail about Operators in Java**

*(You can write this exactly in the exam. Structured, points, tables, examples — nothing missing.)*



## **1. Introduction**

In Java, operators play a **fundamental role** in performing various **computations, decisions, and logical operations**.
An operator is a **symbol that performs an operation** on one or more operands.
Java provides a **rich set of operators** to support arithmetic calculations, relational comparisons, logical decisions, bit manipulation, and assignment operations.



## **2. Definition**

An **operator** in Java is a special symbol that performs a **specific operation** on one or more operands and produces a result.

**Operand**: The value or variable on which the operator acts.

Example:

```java
int c = a + b;
```

Here:

* `+` → operator
* `a`, `b` → operands



## **3. Importance of Operators**

* Perform mathematical calculations
* Enable decision making
* Control program flow
* Support object-oriented programming
* Improve code efficiency and readability



## **4. Classification of Operators in Java**

Java operators are classified into the following types:

1. Arithmetic Operators
2. Unary Operators
3. Relational Operators
4. Logical Operators
5. Assignment Operators
6. Bitwise Operators
7. Shift Operators
8. Conditional (Ternary) Operator
9. `instanceof` Operator



## **5. Arithmetic Operators**

Arithmetic operators are used to perform **mathematical operations**.

| Operator | Operation      |
| -- | -- |
| `+`      | Addition       |
| `-`      | Subtraction    |
| `*`      | Multiplication |
| `/`      | Division       |
| `%`      | Modulus        |

### **Example**

```java
class ArithmeticOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;
        System.out.println("Addition: " + (a + b));
        System.out.println("Subtraction: " + (a - b));
        System.out.println("Multiplication: " + (a * b));
        System.out.println("Division: " + (a / b));
        System.out.println("Modulus: " + (a % b));
    }
}
```

### **Points**

* `/` gives quotient
* `%` gives remainder
* Works on integer and floating types



## **6. Unary Operators**

Unary operators operate on **a single operand**.

| Operator | Meaning     |
| -- | -- |
| `+`      | Unary plus  |
| `-`      | Unary minus |
| `++`     | Increment   |
| `--`     | Decrement   |
| `!`      | Logical NOT |

### **Example**

```java
class UnaryOperators {
    public static void main(String[] args) {
        int x = 5;
        System.out.println("Initial value: " + x);
        System.out.println("Post Increment: " + (x++));
        System.out.println("After Increment: " + x);
        System.out.println("Pre Decrement: " + (--x));
    }
}
```

### **Pre & Post Increment**

* `++a` → increment first, then use
* `a++` → use first, then increment



## **7. Relational Operators**

Relational operators are used to **compare two values** and return a **boolean result**.

| Operator | Meaning               |
| -- |  |
| `==`     | Equal to              |
| `!=`     | Not equal             |
| `>`      | Greater than          |
| `<`      | Less than             |
| `>=`     | Greater than or equal |
| `<=`     | Less than or equal    |

### **Example**

```java
class RelationalOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        System.out.println(a == b);
        System.out.println(a != b);
        System.out.println(a > b);
        System.out.println(a < b);
        System.out.println(a >= b);
        System.out.println(a <= b);
    }
}
```



## **8. Logical Operators**

Logical operators are used to **combine multiple conditions**.

| Operator | Meaning     |   |            |
| -- | -- | - | - |
| `&&`     | Logical AND |   |            |
| `        |             | ` | Logical OR |
| `!`      | Logical NOT |   |            |

### **Example**

```java
class LogicalOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        if (a > 5 && b > 15) {
            System.out.println("Both conditions are true");
        }
        if (a > 15 || b > 15) {
            System.out.println("At least one condition is true");
        }
    }
}
```

### **Short-Circuit Evaluation**

* `&&` stops if first condition is false
* `||` stops if first condition is true



## **9. Assignment Operators**

Assignment operators assign values to variables.

| Operator | Meaning             |
| -- | - |
| `=`      | Assign              |
| `+=`     | Add and assign      |
| `-=`     | Subtract and assign |
| `*=`     | Multiply and assign |
| `/=`     | Divide and assign   |
| `%=`     | Modulus and assign  |

### **Example**

```java
class AssignmentOperators {
    public static void main(String[] args) {
        int a = 10;
        a += 5;
        System.out.println("After += : " + a);
        a -= 3;
        System.out.println("After -= : " + a);
        a *= 2;
        System.out.println("After *= : " + a);
    }
}
```



## **10. Bitwise Operators**

Bitwise operators work on **binary bits**.

| Operator | Operation   |            |
| -- | -- | - |
| `&`      | Bitwise AND |            |
| `        | `           | Bitwise OR |
| `^`      | Bitwise XOR |            |
| `~`      | Bitwise NOT |            |

### **Example**

```java
class BitwiseOperators {
    public static void main(String[] args) {
        int a = 5;
        int b = 3;
        System.out.println(a & b);
        System.out.println(a | b);
        System.out.println(a ^ b);
        System.out.println(~a);
    }
}
```



## **11. Shift Operators**

Shift operators shift bits **left or right**.

| Operator | Meaning              |
| -- | -- |
| `<<`     | Left shift           |
| `>>`     | Right shift          |
| `>>>`    | Unsigned right shift |

### **Example**

```java
class ShiftOperators {
    public static void main(String[] args) {
        int a = 8;
        System.out.println(a << 1);
        System.out.println(a >> 1);
    }
}
```



## **12. Conditional (Ternary) Operator**

It is a **short form of if-else**.

### **Syntax**

```java
condition ? value1 : value2;
```

### **Example**

```java
class TernaryOperator {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        int max = (a > b) ? a : b;
        System.out.println("Maximum value: " + max);
    }
}
```

## **13. Operator Precedence**

Operator precedence defines the **order of execution**.

Order (High → Low):

1. Unary
2. Arithmetic
3. Relational
4. Logical
5. Assignment



## **14. Advantages of Operators**

* Simplify complex expressions
* Improve program efficiency
* Enable logical decision making
* Reduce code length



## **15. Conclusion**

Operators are the **core elements of Java programming**.
They enable computation, comparison, decision making, and control flow.
A proper understanding of Java operators is essential for writing **efficient, readable, and error-free programs**.
