
# PART 1: DATA TRANSFORMATION

### What is Data Transformation?
Data transformation is the process of converting data from its original format or structure into a format that is suitable for analysis, reporting, or use in other systems. This process often involves cleaning, organizing, and adjusting the data to meet specific requirements for downstream applications. Data transformation can include tasks like aggregation, filtering, encoding, and applying mathematical or statistical operations to modify data.

### Data Transformation by Normalization
*   Normalization is a specific type of data transformation that involves adjusting the values of numerical data to a common scale, without distorting differences in the ranges of values.
*   The purpose of normalization is to make sure that features (variables) have comparable scales, which can be important when working with machine learning models or statistical analysis.

**Why Normalize Data?**
*   **Prevent Bias:** In many models, variables with larger ranges or values can dominate the analysis, overshadowing other variables.
*   **Improve Model Performance:** Normalized data allows models to learn more efficiently because they can treat all features with equal importance.

### Common Methods of Normalization:

**1. Min-Max Normalization:** 
This method scales the data to a specific range, usually between 0 and 1.

$$X_{norm} = \frac{X - X_{min}}{X_{max} - X_{min}}$$

*   **X** is the original value.
*   **X_{min}** and **X_{max}** are the minimum and maximum values of the feature in the dataset.

**2. Z-Score Normalization (Standardization):** 
This technique transforms the data so that it has a mean of 0 and a standard deviation of 1.

$$Z = \frac{X - \mu}{\sigma}$$

*   **X** is the original value.
*   **$\mu$** is the mean of the feature.
*   **$\sigma$** is the standard deviation of the feature.

**3. Decimal Scaling:** 
This method normalizes the data by moving the decimal point.

$$X_{norm} = \frac{X}{10^j}$$

*   **j** is chosen such that the maximum absolute value of the transformed data is less than 1.

### Discretization by binning
*   Discretization by binning is a technique used in data preprocessing to transform continuous numerical data into categorical data.
*   This process involves dividing a continuous range of values into a set of intervals or "bins" and then replacing the values in each bin with a representative value (such as the bin's midpoint or the average value of the bin).
*   The purpose is to simplify the data and reduce its granularity, making it easier to analyze, visualize, or use in machine learning models.

### Types of Binning Methods:

**1. Equal-width Binning:**
*   In equal-width binning, the range of the data is divided into intervals (bins) of equal width (size).
*   The width of each bin is calculated by dividing the difference between the maximum and minimum values by the number of bins you want to create.
*   *Example:* For data: `[2, 5, 7, 9, 12, 15, 18, 21]`
    Divide it into 3 bins (equal width):
    *   Bin 1: `[2, 8]`
    *   Bin 2: `[8, 14]`
    *   Bin 3: `[14, 21]`

**2. Equal-frequency Binning:**
*   In equal-frequency binning, each bin contains an equal number of data points, rather than an equal range of values.
*   This ensures that each bin has approximately the same number of elements, which can be useful when dealing with skewed or imbalanced data.
*   *Example:* For data: `[2, 5, 7, 9, 12, 15, 18, 21]`
    If you want 3 bins, you can divide the data into three groups:
    *   Bin 1: `[2, 5, 7]`
    *   Bin 2: `[9, 12, 15]`
    *   Bin 3: `[18, 21]`

**3. Custom Binning:**
*   Custom binning allows you to define your own bin edges based on domain knowledge or the specific nature of the data.
*   *Example:* If you're categorizing ages, you might define the following bins:
    *   Bin 1: 0-18 (Child)
    *   Bin 2: 19-35 (Young Adult)
    *   Bin 3: 36-60 (Adult)
    *   Bin 4: 61+ (Senior)

**4. Cluster-based Binning:**
*   This method groups data into clusters using clustering algorithms like **K-means**. Each cluster represents a bin, and values within each cluster are assigned to the same bin.
*   Cluster-based binning is useful when data naturally forms groups or clusters.
*   *Example:* If you have data points that naturally group into clusters like `[2, 5, 7]`, `[9, 12, 15]`, and `[18, 21]`, each of these could form a bin.

### Advantages and Disadvantages of Binning:
**Advantages:**
*   **Noise Reduction:** Smoothing
*   **Improved Model Performance:** Accuracy
*   **Interpretability:** Simplicity

**Disadvantages:**
*   **Loss of Information:** Generalization
*   **Choice of Number of Bins:** Arbitrary
*   **Sensitive to Outliers:** Distortion

### Concept Hierarchy Generation for Nominal Data

**What is Nominal Data?**
Nominal data is a type of categorical data that represents distinct categories without any inherent order or ranking. In other words, nominal data consists of labels or names that are used to identify categories, but there is no meaningful way to order them.

Key features:
*   **Categories with no order:** The values are simply different from each other, but there’s no ranking.
*   **No mathematical operations:** You can't perform addition/subtraction. You can’t say one is "greater" or "less".
*   **Labels:** Often used to label things in different categories.

**Examples:**
1. **Colors:** Red, Blue, Green, Yellow (No order like "Red > Blue")
2. **Countries:** USA, Canada, Germany, India
3. **Fruits:** Apple, Banana, Cherry, Mango
4. **Gender:** Male, Female, Non-binary
5. **Types of Animals:** Dog, Cat, Elephant, Tiger

**Concept Hierarchy Generation** 
Refers to grouping those categories into higher-level concepts or abstractions to generalize and simplify data.

#### Concept Hierarchy Example:

**1. Original Nominal Data:**
```text
Animal
------
Dog
Cat
Elephant
Tiger
```

**2. Generated Concept Hierarchy (Grouping by Categories):**
```text
Animal
├── Mammals
│   ├── Dog
│   ├── Cat
├── Wild Animals
│   ├── Elephant
│   └── Tiger
```

**3. Generalized Data:**
```text
Generalized Animal
-------------------
Mammals
Mammals
Wild Animals
Wild Animals
```

**Why is Concept Hierarchy Important?**
*   **Simplifies the Data:** Reduces complexity by grouping detailed categories into broader concepts.
*   **Improves Understanding:** Makes it easier to understand patterns/trends at a higher level.
*   **Data Mining Efficiency:** Algorithms work more efficiently handling fewer broad categories.
*   **Better Insights:** Helps see relationships that might not be obvious with individual categories.

# PART 2: DATA REDUCTION

### Data Reduction
It refers to the process of minimizing the volume of data while retaining its essential characteristics or insights. This is crucial in data analysis, machine learning, and storage management, as it helps to simplify datasets, reduce computational load, and improve processing efficiency without losing significant information.

**Key Techniques:**
*   **Dimensionality Reduction:** Reducing the number of features or variables in the dataset.
    *   *Examples:* 1. Principal Component Analysis (PCA) | 2. Attribute Subset Selection

### What Is Principal Component Analysis (PCA)?
Principal components analysis (PCA) is a dimensionality reduction technique that enables you to identify correlations and patterns in a data set so that it can be transformed into a data set of significantly lower dimension without loss of any important information.

#### Steps to Compute Principal Components (Flowchart):
```text
[Data Acquisition] ➔ [Standardization of Dataset] ➔ [Collection of Covariance Matrix] ➔ [Calculation of Eigenvalue and Eigen Vector]
                                                                                                        |
                                                                                                        ▼
                                                          [Deriving new dataset] 🡄 [Choosing Components and forming a feature vector]
```

### PCA Mathematical Example:
**Example Data:** Let’s say we have a dataset of 2D points (Each row represents a data point with two features):

$$ X = \begin{bmatrix} 2 & 3 \\ 3 & 3 \\ 4 & 3 \\ 5 & 4 \\ 6 & 5 \end{bmatrix} $$

**Step 1: Mean-Centering the Data**
Calculate the mean of each feature and subtract it from the corresponding feature value.
*   Mean of feature 1:  $$\mu_1 = \frac{2+3+4+5+6}{5} = 4$$
*   Mean of feature 2:  $$\mu_2 = \frac{3+3+3+4+5}{5} = 3.6$$

Subtract the means from the dataset to center it:
$$ X_{centered} = \begin{bmatrix} 2 - 4 & 3 - 3.6 \\ 3 - 4 & 3 - 3.6 \\ 4 - 4 & 3 - 3.6 \\ 5 - 4 & 4 - 3.6 \\ 6 - 4 & 5 - 3.6 \end{bmatrix} = \begin{bmatrix} -2 & -0.6 \\ -1 & -0.6 \\ 0 & -0.6 \\ 1 & 0.4 \\ 2 & 1.4 \end{bmatrix} $$

**Step 2: Compute the Covariance Matrix**
$$ Cov(X) = \frac{1}{n-1} X_{centered}^T X_{centered} $$
Where $n = 5$.

1. Transpose $X_{centered}$:
$$ X_{centered}^T = \begin{bmatrix} -2 & -1 & 0 & 1 & 2 \\ -0.6 & -0.6 & -0.6 & 0.4 & 1.4 \end{bmatrix} $$

2. Compute $X_{centered}^T X_{centered}$:
$$ X_{centered}^T X_{centered} = \begin{bmatrix} 10 & 5.0 \\ 5.0 & 3.2 \end{bmatrix} $$

3. Dividing by $n-1 = 4$:
$$ C = \frac{1}{4} \begin{bmatrix} 10 & 5.0 \\ 5.0 & 3.2 \end{bmatrix} = \begin{bmatrix} 2.5 & 1.25 \\ 1.25 & 0.8 \end{bmatrix} $$

**Step 3: Eigenvalue Equation**
To find eigenvalues ($\lambda$), we solve the characteristic equation $\det(C - \lambda I) = 0$.

$$ C - \lambda I = \begin{bmatrix} 2.5 - \lambda & 1.25 \\ 1.25 & 0.8 - \lambda \end{bmatrix} $$

Determinant:
$$ \det(C - \lambda I) = (2.5 - \lambda)(0.8 - \lambda) - (1.25)(1.25) = 0 $$
$$ \lambda^2 - 3.3\lambda + 0.4375 = 0 $$

Solving the quadratic equation ($a=1, b=-3.3, c=0.4375$):
$$ \lambda = \frac{-(-3.3) \pm \sqrt{(-3.3)^2 - 4(1)(0.4375)}}{2(1)} = \frac{3.3 \pm \sqrt{10.89 - 1.75}}{2} = \frac{3.3 \pm \sqrt{9.14}}{2} $$
Here, $\sqrt{9.14} \approx 3.02$.
*   $\lambda_1 = 3.162$
*   $\lambda_2 = 0.14$

**Eigenvectors:**
For $\lambda_1 \approx 3.1615$:
$$ (C - 3.1615I)v = 0 \implies \begin{bmatrix} -0.6615 & 1.25 \\ 1.25 & -2.3615 \end{bmatrix} \begin{bmatrix} v_1 \\ v_2 \end{bmatrix} = 0 $$
Solving gives eigenvector $\approx \begin{bmatrix} 0.883 \\ 0.467 \end{bmatrix}$

For $\lambda_2 \approx 0.1385$:
$$ (C - 0.1385I)v = 0 \implies \begin{bmatrix} 2.3615 & 1.25 \\ 1.25 & 0.6615 \end{bmatrix} \begin{bmatrix} v_1 \\ v_2 \end{bmatrix} = 0 $$
Solving gives eigenvector $\approx \begin{bmatrix} -0.472 \\ 0.882 \end{bmatrix}$

**Final Principal Components:**
*   **PC1:** $\begin{bmatrix} 0.883 \\ 0.467 \end{bmatrix}$ *(explains most variance)*
*   **PC2:** $\begin{bmatrix} -0.472 \\ 0.882 \end{bmatrix}$ *(explains remaining variance)*

### Attribute Subset Selection
A technique used to pick the most important features from a dataset, while removing the irrelevant, redundant, or noisy ones to make the model simpler, faster, and more accurate.

**Why is it Important?**
*   **Improves model accuracy:** Focuses on important features.
*   **Reduces computation time:** Less data to process.
*   **Prevents overfitting:** Avoids models getting "too specific" to training data.

**Methods:**
1.  **Stepwise Forward Selection:** Start with empty set. Add the most important feature one by one until no further improvement.
2.  **Stepwise Backward Elimination:** Start with all features. Remove the least important one by one until performance decreases.
3.  **Combination of Forward and Backward:** Mix both adding useful features and removing unhelpful ones.
4.  **Decision Tree Induction:** Tree automatically selects the best features to split data based on categories.

### Attribute Subset Selection Measures

**1. Correlation Coefficient:** 
Measures strength and direction of relationship between two numerical variables. 
*   *Pearson's:* Linear correlation for continuous features.
*   *Spearman's:* Ordinal or non-linear data.

**2. Information Gain (IG):** 
Used in decision trees to split data, measuring how much uncertainty/entropy is reduced.

$$ Information\ Gain(IG) = Entropy(S) - \sum_{i=1}^k \frac{|S_i|}{|S|} \cdot Entropy(S_i) $$

**Entropy Formula:**
$$ Entropy(S) = - \sum_{i=1}^n p_i \log_2 p_i $$
*(Where $p_i$ is probability of class $i$, $n$ is number of classes. Entropy ranges from 0 to 1).*

**3. Chi-Square Test:** 
Statistical test to see if there is a significant relationship between two categorical variables.

$$ \chi^2 = \sum \frac{(O - E)^2}{E} $$
*(Where $O$ is observed value, $E$ is expected value. High value means feature is likely related to the target).*

# PART 3: DATA PRE-PROCESSING

### Overview of Data Preprocessing
*   Data preprocessing is a crucial step in the data analysis pipeline, especially when working with machine learning (ML) and data science tasks. It refers to the techniques and processes used to clean and transform raw data into a suitable format.
*   In raw form, data often contains inconsistencies, missing values, noise, and other issues. Preprocessing ensures the model performs optimally.

### Data Quality in Data Preprocessing
1.  **Accuracy:** Data should correctly represent the real-world situation.
2.  **Completeness:** Data should not be missing values.
3.  **Consistency:** Data must be consistent across datasets.
4.  **Timeliness:** Data should be up-to-date.
5.  **Relevance:** Pertinent to the research/analysis.
6.  **Uniformity:** Standardized units and formats.

### Major Tasks in Data Preprocessing
*   **Data Cleaning:** Handle missing values, remove duplicates, correct inconsistencies.
*   **Data Integration:** Combine data from multiple sources, resolve schema conflicts.
*   **Data Transformation:** Normalize/scale data, encode categorical variables, generate features.
*   **Data Reduction:** Dimensionality reduction, sampling.
*   **Data Encoding:** Convert data types to numerical formats.
*   **Data Splitting:** Split into training, validation, and testing sets.
*   **Outlier Detection:** Identify and treat anomalies.
*   **Data Balancing:** Address imbalanced datasets (oversampling/undersampling).

### Data Cleaning
**1. Missing Values (When Data is Missing)**
*   *Causes:* Not collected properly, forgotten form entries, data source errors.
*   *Handling:* Delete the data row, or fill it using Mean, Mode, or a Prediction model.

**2. Noisy Data (When Data is Messy or Incorrect)**
*   *Causes:* Mistyped data, faulty sensors, outliers.
*   *Handling:* Remove outliers, smooth the data (averaging), fix known errors, or normalize to a similar range.

#### Key Differences Between Missing Values and Noisy Data

| | Missing Values | Noisy Data |
| :--- | :--- | :--- |
| **Definition** | Some data is **missing** (empty). | Data is **incorrect, too high, too low,** or **not relevant.** |
| **Cause** | Happens when data isn't recorded or collected. | Happens due to errors, mistakes, or extreme values. |
| **Action** | You try to **fill** the missing values or **remove** incomplete records. | You try to **fix** errors or **remove** incorrect data points. |
| **Example** | Empty age in student data. | A student's age recorded as 500. |

### Data Integration & Entity Identification
Data integration is like combining different puzzle pieces to create a complete picture. **Entity identification** (or resolution) is about finding out which records in different datasets refer to the same real-world object (e.g., "John Smith, 123 Main St." vs "J. Smith, Main Street, NY").

*   **Importance:** Improves data quality, efficiency, decision-making, and compliance/reporting.
*   **Steps:** 
    1. Data Collection
    2. Attribute Matching
    3. Use Algorithms to Score Similarity
    4. Decide and Link (based on a threshold).
*   **Tools:** Rule-based approaches, Machine Learning, Libraries like Python's `fuzzywuzzy`.

### Redundancy and Correlation Analysis in Tuple Duplication
A **tuple** is a single record/row in a database. Tuple duplication occurs when multiple tuples contain identical or nearly identical data.

**1. Redundancy Analysis:**
*   **Exact Duplication:** Every attribute matches perfectly.
*   **Near Duplication:** Minor differences (typos). Addressed via fuzzy matching or approximate string matching (e.g., Levenshtein distance).

**2. Correlation Analysis:**
Examines relationships between attributes to see if they refer to the same entity despite minor differences.
*   **High Correlation:** Highly likely to be the same entity (treated as duplicate).
*   **Low Correlation:** Likely different entities.

**Methods:** Exact matching, Fuzzy matching, Threshold-Based matching, Clustering, Data Normalization, Correlation Metrics.

### Data Value Conflict Detection and Resolution
Conflicts arise when there are discrepancies between different sources (e.g., duplicate data, inconsistent data, missing data, or conflicting facts).

**Steps to Resolve:**
1.  **Identify the Conflict:** Use validation tools.
2.  **Investigate the Cause:** Human error, system bug?
3.  **Choose a Resolution Method:** Manual correction, automated rules, data standardization, or prioritizing the most reliable source.
4.  **Consistency Rules:** Set rules to avoid future issues.
5.  **Review and Verify:** Double-check accuracy.

**Best Practices:** Follow data entry standards, perform regular audits, use database rules, and ensure staff training.
